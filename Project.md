# 背景

我要做一个家族樹的應用程式，可以顯示家族成員之間的關係，並且可以編輯家族成員的資訊。
應用程式的基型式不限，可以是網頁應用程式，也可以是桌面應用程式，也可以是行動應用程式。使用的程式語言不限，但是必須是開源的。

# 需求
## 概念

這個應用程式必須要滿足以下需求：
1. 它可以完整的顯示一個從一對夫妻開始的所有後代成員以及他們間的關係，其中包含以下關係
    - 夫妻：並列在左右
    - 子女：與父母連接後並列在父母下方
2. 同一對夫妻的子女依年紀大至小由左至右排列

## 操作

1. 可以在一個圖形介面上新增，刪除，編輯每個成員的資訊。

# 畫面呈現

# 畫面呈現

我希望有幾種不同的視圖。這些視圖必須是可以自由切換的：
1. **一般的家庭樹**
2. **有出生時間關係的家庭樹 (垂直對齊)**：在一般家庭樹的基礎上，把每個同年出生的人調整到垂直方向的相同水平線上。即使跨越原本的垂直世代連線，也直接將從父母拉出來的關係線拉長。
3. **有出生時間關係的家庭樹 (極座標表示)**：中心是最年長的人，向外的每個同心圓都是同一個年紀（或是同一個世代）的人。
4. **介面操作**：支援縮放與拖曳 (Zoom in/out, Pan) 以應對龐大樹狀結構。

# 系統規格

1. **技術選型**：採用 Web Application，並考量不需後端環境，未來能以靜態網頁 (如 Github Pages) 方式部署。
2. **資料儲存**：單純在前端運作，支援資料匯出與匯入功能 (例如 JSON 檔案)。
3. **邊界情況與操作細節**：
    - 不需支援多配偶或非親生子女等複雜關係。
    - 頂層的起始夫妻，其父母欄位可為空。
    - 樹狀圖只往下繪製子孫代 (暫不支援往上繪製祖父母，但底層資料結構可保留彈性)。
    - **配偶連線**：一對配偶間只會有一人是家族直系血親（除非是頂層根節點），父母的連線只需連到該直系子女即可，配偶再與之水平相連。
    - 若子女為單身，畫面上只顯示自己。
    - 編輯介面採用側邊欄 (Sidebar) 進行新增、刪除、編輯成員資訊。
4. **成員欄位**：
    - 必填：姓名、性別、出生時間、父母、配偶。
    - 選填：忌日 (若已去世)、照片 (透過圖片 URL 提供)、備註內容。
